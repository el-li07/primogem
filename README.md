# Primogem Auth Library
![Python](https://img.shields.io/badge/python-3.10+-blue.svg)
![FastAPI](https://img.shields.io/badge/FastAPI-0.100+-05998b.svg)
![JWT](https://img.shields.io/badge/JWT-EdDSA-orange.svg)
![License](https://img.shields.io/badge/license-MIT-green.svg)

**Primogem** - это легковесная и простая в использовании библиотека для аутентификации, построенная на базе **FastAPI** и **JWT**, разработанная специально для корпоративных систем с акцентом на безопасность, простоту интеграции и удобство использования. Основная особенность - использование современного алгоритма подписи **EdDSA** и встроенная система автоматической ротации ключей и проверки токенов. 
## Основные возможности
- Подпись и верификация JWT-токенов с помощью EdDSA
- Автоматическая ротация ключей с архивацией старых
- Шифрование приватных ключей
- Гибкая система ролей и scopes
- Готовый сервер авторизации на FastAPI
## Установка
Для установки клонируйте репозиторий:
```bash
git clone https://github.com/el-li07/primogem.git
cd primogem
```
Установите зависимости:
```bash
pip install -r requirements.txt
```
После чего установите библиотеку в режиме разработки (рекомендуется):
```bash
pip install -e .
```
## Быстрый старт
### 1. Первичная настройка
Запустите мастер настройки, чтобы сконфигурировать пути и настроить защиту системы:
```
python -m primogem.cli setup
```
В процессе настройки можно задать название организации, путь для сохранения ключей и базы данных пользователей.
### 2. Создание пользоателей
Для создания пользователей воспользуйтесь следующей командой:
```
python -m primogem.cli create_user username password "full_name" "department" --roles --scopes
```
### 3. Запуск Auth Server
Библиотека предоставляет готовый сервер для выдачи и проверки токенов. Воспользуйтесь следующей командой:
```
python -m primogem run-server
```
## Команды
### 1. setup
Производит первичную настройку системы. В ходе настройки задаются следующие параметры:
- *Название организации*. По умолчанию устанавливает название "Моя Компания".
- *Путь для хранения ключей и базы данных*. По умолчанию создает папку для ключей и базу данных в корневой папке библиотеки.
- *Мастер-пароль*. По умолчанию создает систему без пароля. Обратите внимание, что это не является безопасным методом хранения парольно-ключевой информации, и система может быть скомпроментирована.
### 2. config
Позволяет настроить сроки жизни и хранения сгенерированных ключей. Для настройки используйте следующие флаги:
- `--rotation` - настроить сроки ротации активных ключей в днях.
- `--retention` - настроить сроки хранения заархивированных ключей в днях.
### 3. set-path
Позволяет настроить директорию для сохранения базы данных пользователей и сгенерированных ключей. Принимает значение в формате `set-path path`, где *path* - новый путь.
Обратите внимание, что после перенастройки данные не переносятся самостоятельно.
### 4. create-user
Создает нового пользователя в базе денных. Назначения переменных и флагов:
- `username` - логин пользователя;
- `password` - пароль пользователя;
- `"full_name"` - ФИО пользователя (или допустимый аналог);
- `"department"` - отдел пользователя;
- `-- roles` - роль/и пользователя. По умолчанию присваивается значение `"employee"`;
- `-- scopes` - области доступа пользователя в системе. Указывается в формате `object:permission`. Допустимо указывать несколько областей доступа внутри одной функции через запятую, без пробелов. По умолчанию присваивается значение `"files:read"`.
### 5. run-server
Запускает сервер генерации токенов и аутентификации. Развернутый сервер будет находиться по адресу http://127.0.0.1:8000. Документацию Swagger можно найти по адресу http://127.0.0.1:8000/docs.
### 6. rotate-key
Выполняет принудительную ротацию текущего ключа независимо от того, истек срок действия или нет. Обновленный ключ будет сохранен в архиве.
## Структура директории
При инициализации библиотека создает следующую структуру по указанному в настройках пути:
- `primogem.db` - SQLite база данных пользователей.
- `keys/`
  - `active_keys.json` - текущие пары ключей.
  - `archive/` - архив старых приватных ключей для проверки ранее выданных токенов.
## Файл конфигурации
В Primogem используется файл конфигурации `.env`, отвечающий за хранение некоторых нараметов системы. С его содержимым по умолчанию вы можете ознакомиться ниже:
| Переменная | Тип | Описание | По умолчанию |
| :--- | :---: | :--- | :--- |
| `CORS_ORIGINS` | `list` | Список разрешенных адресов для доступа к API (через запятую). | `["http://localhost:5500", "http://127.0.0.1:5500"]` |
| `BASE_PATH` | `str` | Директория для хранения базы данных и папки с ключами. | `.` |
| `KEY_ROTATION_DAYS` | `int` | Период в днях, по истечении которого система создаст новую пару ключей. | `30` |
| `KEY_ARCHIVE_RETENTION_DAYS` | `int` | Сколько дней хранить старые ключи в архиве перед окончательным удалением. | `90` |
## Рекомендации по интеграции
### Логирование для аудита
Чтобы понимать, что происходит внутри, настройте стандартный логгер Python. Primogem использует его для всех важных событий.
### Настройка CORS
В файле `.env` вы можете указать адреса фронтенда, если браузер блокирует запросы к `/login`. По умолчанию библиотека использует `CORS_ORIGINS=["http://localhost:5500", "http://127.0.0.1:5500"]`.
### Автоматизация ротации ключей
Безопасность требует регулярной смены ключей. В Primogem этот процесс автоматизирован, но его нужно вызвать в фоновом режиме, например, при запуске сервера или при помощи FastAPI lifespan.
```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from primogem.key_manager import KeyManager

@asynccontextmanager
async def lifespan(app: FastAPI):
    km = KeyManager()
    km.check_and_rotate_if_needed()
    yield

app = FastAPI(lifespan=lifespan)
```
### Разграничение прав
Библиотека поддерживает проверку ролей и конкретных областей доступа. Используйте `require_role` и `require_scope`, чтобы легко интегрировать проверки в свой сервис.
```python
from primogem.dependencies import require_role, require_scope

# Доступ только для администраторов
@app.get("/admin/users")
async def list_users(admin = Depends(require_role("admin"))):
    return {"users": [...]}

# Доступ только при наличии определенного scope в токене
@app.post("/reports/export")
async def export_report(user = Depends(require_scope("reports:export"))):
    return {"status": "export_started"}
```
### Проверка токена
Класс `TokenVerifier`- это основной инструмент для ваших микросервисов. Вы можете настроить его поведение под конкретные задачи.
```python
from primogem import TokenVerifier

verifier = TokenVerifier()

# Пример: проверка токена с извлечением данных
try:
    payload = verifier.verify(token)
    
    user_dept = payload.get("department")
    user_roles = payload.get("roles", [])
    
    if "admin" in user_roles:
        print(f"Админ из отдела {user_dept} вошел в систему")
        
except ValueError as e:
    print(f"Критическая ошибка безопасности: {e}")
```
### Ручная проверка токена
Если вам нужно проверить токен вне контекста FastAPI, например, в фоновой задаче или скрипте, вы также можете использовать `TokenVerifier`.
```python
from primogem import TokenVerifier

verifier = TokenVerifier()

def process_background_task(token: str):
    try:
        payload = verifier.verify(token)
        print(f"Обработка задачи для пользователя: {payload['username']}")
    except ValueError as e:
        print(f"Ошибка: {e}")
```
### Интеграция через JWKS
Если ваш сервис написан на другом языке (Go, Java, Node.js) или вы используете сторонний API Gateway, вы можете настроить проверку через стандартный эндпоинт JWKS.
  URL: `http://auth-server:8000/.well-known/jwks.json`
  Алгоритм: `EdDSA`
  Кривая: `Ed25519`
При ротации ключей в Primogem список публичных ключей по этому адресу обновится автоматически. Ваш сервис-потребитель должен кэшировать эти ключи и проверять подпись входящих JWT.
## Тестирование
Проект имеет набор тестов для проверки корректности работы криптографических функций, логики ротации ключей и защиты от распространенных атак на JWT.
### Запуск тестов
Для запуска используется фреймворк `pytest`. Так как система использует шифрование ключей, перед запуском необходимо задать пароль в переменных окружения:
```bash
# Переменная окружения для Linux/macOS
export PYTEST_ENCRYPTION_PASSWORD=[ваш_пароль]

# Переменная окружения для Windows
set PYTEST_ENCRYPTION_PASSWORD=[ваш_пароль]
```
### Описание тестовых модулей
| Файл | Что тестируется |
| :--- | :--- |
| `test_security.py` | Защита от атак None algorithm и Algorithm substitution, проверка Expired и Invalid токенов. |
| `test_key_manager.py` | Генерация пар Ed25519, шифрование приватных ключей паролем и механизм ротации. |
| `test_verifier.py` | Корректность работы `TokenVerifier`: поиск нужного ключа в хранилище и декодирование Payload. |
| `test_database.py` | Работа с SQLite: создание пользователей, хеширование паролей через `bcrypt` и поиск по `sub`. |
| `test_config.py` | Валидация настроек Pydantic: загрузка переменных из `.env` и корректность значений по умолчанию. |
| `conftest.py` | Автоматическая настройка окружения, инициализация тестовой БД и проброс мастер-пароля. |
## Демонстрационное приложение
В папке `demo-server` находится пример использования библиотеки - веб-портал для управления документами организации. Демо-сервис демонстрирует работу ролевой модели доступа и областей доступа в связке с асимметричной верификацией токенов. Демо-сервер выступает в роли ресурсного сервера. Он не знает паролей пользователей, и доверяет только JWT-токенам, подписанным сервером авторизации. Сервис использует `TokenVerifier` для проверки подписи EdDSA. На основе данных из токена (sub, roles, department) применяются функции `can_read` и `can_edit`.

### Настройка демо-сервера
Используйте команду `python -m primogem.cli setup` и введите следующие данные:
- Название организации: `test_company`
- Путь для хранения ключей и базы данных: `demo-server`
Остальные параметры можете настроить на свое усмотрение.
### Запуск демонстрации
1. Запустите сервер авторизации: `python -m primogem.cli run-server`;
2. Запустите демонстрационный сервер: `python -m demo-server.main`;
3. Запустите фронтэнд (опционально): `python -m http.server 5500`.

Демонстрационный сервер будет доступен по адресу http://localhost:8001/, фронтэнд -- по адресу http://localhost:5500/demo-server.
